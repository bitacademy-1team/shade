데이터 불러오는 작업
justwatch.com 에서 비공식적으로 지원해주는 api 를 이용하여
영화,티비쇼에 관련된 정보를 json 형식으로 받아온 데이터를 java에서 simple json 을 이용하여
사용하는 db에 넣었다

영화 줄거리에서 키워드를 추출하여(Twitter,kkma사용)
키워드,장르,타이틀,배우,감독의 데이터로 영화를 추천할 수 있었다

sikitrean,suprise,numpy,pandas등 을 사용하여 영화 추천 시스템 구축

각 영화마다 일일이 영화 평점(1~5점)을 주는것보단 좋아요/싫어요 로 단순하게 구성하고 싶었으나
레퍼런스로 많이 나와있는 추천시스템은(suprise을 활용한) 평점이 꼭 필요 하였다
처음에는 좋아요(1점)/싫어요(0.1)점으로 평점을 주어 시스템을 구성하려 했으나 정확도가 높지않은걸
확인, 그 후에 shade만의 레이팅 시스템(최근 평가한 영화를 바탕으로 높은평점을 준 장르,주요 배우,감독
으로 가중치 적용)을 사용하여 사용자에게 보이지않는 시스템 내부적인 평점을 만들어서 
추천시스템에 적용 하였다.

우리가 사용하고자 하는 추천 시스템은 대부분 java보단 python을 사용하는게 일반적이라서
python을 사용하기로 해서 django를 이용하여 python이 필요한 부분을 개발 하게 되었다

django를 사용하던 중 django에서 편리하게 관리자페이지를 제공한다는것을 알게되었고
그 기능을 사용하기로 했다

jpa
간단한것들은 jpa의 기본 기능으로 구현하였고 
복잡한 쿼리문이 필요할때 native 쿼리를 이용하여 구현하였다
(one to many, many to one 같은 데이터 관계설정 미숙이 아쉽다)

querydsl 사용
동적인 쿼리가 필요하게되어 찾아보다가 querydsl 이란걸 알게 되어서
querydsl의 PredicateQuery 기능으로 동적인 쿼리 기능을 구현하였다
